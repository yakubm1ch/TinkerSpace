# Contributing to TinkerSpace

Thank you for contributing!

## Branch Strategy

- `main` — stable branch
- `feature/*` — new features
- `fix/*` — bug fixes

Never develop directly on `main`.

---

## Commit Messages

Examples:

- feat: add admin dashboard
- fix: resolve router issue
- docs: update roadmap
- refactor: improve project structure

---

## Code Style

- Keep components small and reusable.
- Use meaningful names.
- Write readable code.
- Avoid unnecessary complexity.

---

## Documentation

When adding a major feature, update:

- ROADMAP.md
- PROJECT_VISION.md (if needed)
- CHANGELOG.md

---

## Goal

Build something we can be proud of.
